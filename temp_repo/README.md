## 项目简介

**VisionZone (项目代码: CVZJ - Computer Vision Zone Journey)** 是一项旨在为视障人士提供“赛博视觉皮层”的前沿 AI 开源项目。不同于传统的依靠超声波避障或简单物体识别的辅助设备，VisionZone 融合了计算机视觉（CV）领域最尖端的实时深度估计、场景语义理解和实例分割技术，结合大语言模型（LLM）的常识理解能力，构建了一个具备高度空间意识和自然交互能力的导航大脑。

我们的目标是打造一个能够穿透迷雾、感知立体世界、并以“类人”语气提供播报的智能辅助系统，让每一位视障人士都能享受到技术进步带来的出行尊严与自由。

## 核心理念与社会价值

### 1. 感知平等 (Equality of Perception)
我们认为，对世界的感知力不应受限于生理条件。VisionZone 通过深度学习算法将复杂的 3D 视觉空间数字化，再通过语义化播报将信息传递给用户，从而在算法层面弥补了视觉缺失。

### 2. 自主与尊严 (Autonomy & Dignity)
传统的盲杖和导盲犬虽然有效，但往往让用户成为人群中的焦点。VisionZone 致力于小型化与隐形化，通过智能穿戴设备（如 AR 眼镜或肩挎式摄像头）实现低调导航，让用户重新获得独立探索世界的信心。

### 3. 算法无偏见 (Zero Judgment)
VisionZone 的核心算法基于纯粹的几何物理规律。它不会因为场景的繁杂而感到疲惫，也不会对环境产生主观偏差。每一毫米的距离计算、每一类场景的划分，都是冷峻而忠实的。

## 系统架构与技术选型

VisionZone 采用了 **“感知-决策-执行-交互”** 的四层分布式架构，确保系统在处理海量图像数据的同时，依然能保持低延迟和高鲁棒性。

### 感知层：多模态视觉融合
这是系统的“眼睛”，集成了当前学术界性能最优的三个模型：
- **Depth Anything V2 (Small/Base)**: 负责构建亚像素级的深度图。它无需双目摄像头，仅凭单帧图像即可恢复场景的相对远近，解决“那里有什么”之外的“那里有多远”的问题。
- **Mask2Former (Swin-Tiny)**: 基于 Transformers 的通用分割框架。它可以同时处理语义分割、实例分割和全景分割。我们主要利用其在 ADE20K 数据集上的预训练能力，精准识别出地板、人行道、草地、天空和建筑。
- **YOLOv26-SEG**: 作为极速实例分割模型，它专门负责识别具有威胁性的动态目标（如行人、车辆、施工围栏），并提供像素级的精确轮廓掩膜（Mask）。

### 寻路层：物理路径规划
这是系统的“小脑”，负责将感知数据转化为具体的行走建议：
- **安全掩膜提取**: 系统会自动识别出场景中的“Free Space”（可通行区域），并将盲道（黄色触感板）作为高优先级权重叠加。
- **递归中轴算法**: 通过对可通行区域进行水平切片，计算每一层的重心，连成一条适应地形的“虚拟路径”。

### 交互层：大语言模型（LLM）驱动
这是系统的“嘴巴”和“社交大脑”：
- **Gemma 3N / Llama 3**: 通过端侧部署的 Ollama 引擎，将复杂的传感器参数（偏航角 15.4度、前方 3.2米有卡车、环境是雨天）转化为温柔、简洁、具备人性化关怀的语音指令。

## 核心算法深度解析

### 单目深度估计与距离标定
传统的单目视觉很难准确计算距离。VisionZone 引入了一个**实验性逆深度标定公式**，将 `Depth Anything V2` 输出的 0-255 相对深度转化为物理距离 $d$（单位：米）：

$$d = K \times \frac{1.05 - \text{depth}_{norm}}{\text{depth}_{norm} + 0.05}$$

其中 $K$ 是针对不同焦距摄像头校准的常数。通过提取 YOLO 掩膜区域内的深度中位数，我们可以精准获知前方行人的具体步数，而不仅是简单的“远”或“近”。

### 全景语义分割与环境自适应
系统通过分析 `Mask2Former` 的输出像素占比来自动切换模式：
- **室内模式**: 当检测到“Ceiling（天花板）”像素超过阈值且“Sky（天空）”像素接近零时，系统自动切换为针对平滑地板的避障参数，避让家具、楼梯口。
- **室外模式**: 当检测到“Road（道路）”或“Sidewalk（人行道）”时，系统会重点加强对车辆和交通灯的识别，并启动“斑马线寻找”算法。

### 递归切片路径规划算法
为了计算最佳行走路径，系统在图像的 $0.5H$ 到 $0.9H$（用户脚下区域）设置了多个探测切片。
1. **行分析**: 对每一行像素提取所有的可通行点。
2. **重心计算**: 计算该行安全像素的平均 X 坐标，得到该层的“路径中心点”。
3. **向量加权**: 将多层中心点进行平滑拟合，利用 `math.atan2` 计算出相对于用户轴线的偏航角（Yaw Angle）。这使得系统能够引导用户平滑地绕过障碍物，而不是突兀的转向。

### 动态物体避障：像素级“黑洞”策略
VisionZone 不再使用矩形框避障。利用 YOLO26-SEG 的 Mask 输出，我们将物体所在的确切像素从“安全掩膜”中剔除。这意味着，即使用户在狭窄的小巷遇到形状不规则的电瓶车，系统也能利用像素级的缝隙计算出最优的一条细细的通路。

## 可视化系统：开发者 HUD 与 AR 增强

虽然视障用户不需要屏幕，但在研发和调试阶段，一个高精度的 **FSD-Style HUD (飞行员模拟现实界面)** 至关重要。

`HUDVisualizer` 模块实现的功能包括：
- **三连屏渲染**: 同时展示深度感知流、语义理解流和最终的 AR 增强流。
- **安全区域热力图**: 以绿色半透明遮罩叠加在原始画面上，实时展示算法眼中的“康庄大道”。
- **动态风险预警**: 当障碍物进入 3 米半径内时，Mask 颜色会从橙色变为深红，并闪烁边框。
- **中文支持**: 全面支持 UTF-8 中文渲染，展示语音播报生成的“思考过程”。

## 大模型交互设计（Prompt Engineering）

我们认为，冷冰冰的数字（如“左转 15 度”）对视障用户是不友好的。`VoiceAssistant` 模块通过以下 Prompt 策略将数据转化为温度：

> **输入数据**: `环境：室外人行道` | `障碍物：汽车(1.5米, 正前方)` | `建议：向右避让` | `偏角：20度`
>
> **大模型重塑结果**: "⚠ 请注意，正前方 1.5 米停靠了一辆汽车。别担心，请跟我向右侧稍微迈出一小步，前方路就宽了。"

这种重塑过程由本地部署的 **Gemma 3N** 完成，确保了极高的隐私保护，用户的视觉数据不会上传到任何云端服务器。

## 安装与运行指南

### 1. 硬件环境要求
- **GPU**: NVIDIA RTX 3060 或以上 (8GB+ 显存)。
- **CPU**: 4 核以上。
- **摄像头**: 广角 1080P USB 摄像头或 CSI 摄像头。
- **架构**: 支持 Windows (WSL2), Linux (Ubuntu 22.04+)。

### 2. 软件依赖安装
```bash
# 创建虚拟环境
conda create -n cvzj python=3.10
conda activate cvzj

# 安装核心依赖
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu118
pip install transformers ultralytics opencv-python matplotlib ollama pillow
```

### 3. 模型准备
1. 下载 `yolo26n-seg.pt` 权重并放置在根目录。
2. 启动 [Ollama](https://ollama.com/) 并拉取模型：
   ```bash
   ollama pull gemma3n
   ```

### 4. 快速启动
```bash
python main.py
```

## 未来路线图：从感知到认知

VisionZone 的旅程才刚刚开始。我们接下来的三个阶段规划：

- **V2 阶段 (2026 Q3)**:
    - 引入多模态大模型 (如 LLaVA/Gemma-Vision)，实现对场景的复杂细节理解（例如：“前面那个路标写着超市”。）
    - 结合 IMU 传感器实现六自由度（6-DoF）的空间定位，解决上下坡识别难题。
- **V3 阶段 (2027)**:
    - 协同导航：实现多个设备间的信息共享，预警盲区外的车辆。
    - 低功耗芯片移植：针对边缘计算平台（如 Jetson Orin）进行 FP16/INT8 量化压缩。
- **Vα 阶段 (终极形态)**:
    - 实现全天候、全场景、零干预的完全自主导航，真正成为视障人士的“数字生命”伴侣。
